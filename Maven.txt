Maven:
1. Tornar processo de build facil
2. Prover uniforme sistema de build
3. Prover informacao do projeto de qualidade
4. Prover guias de boas praticas desenvolvimento
5. Pertmitir transparente migracao a novas ferramentas/funcionalidades

> Criar projeto com conjunto de padroes definidos e gerenciar o ciclo de vida e dependencias do projeto

> Estrutura de projeto padrao
> POM ( Project Object Model) 

> Ciclo de vida do projeto
	> clean: prepara o projeto para o build removendo arquivos desnecessarios e dependencias
	> default: builda o projeto
	> site: cria documentacao do projeto
> Maven divide esses ciclo de vidas em fases que representa um estágio no processo de build
> Default phases:
	> Validate
	> Compile	
		> compile
		> testCompile
	> Test
	> Package
	> Deploy

---

> Gerenciamento de dependencias
	> dependencia é um 'artifact' que o software requer/precisa

> <scope> na dependencia
> 5 scopes comuns:
	> compile: a dependencia estará presente em todos os classpaths e propagada a projetos dependentes
	> provided: igual ao compile, mas a dependencia é esperada ser provida em tempo de execução pelo ambiente de execução (JDK) ou web container
	> runtime: a dependencia não é requerida em tempo de compilação mas em tempo de execução
	> test:  a dependencia somente é requerida enquanto compilando e executando testes e não estará presente no classpath durante normal execução ou propagada a projetos dependentes
	> system: igual ao compile, mas o caminho da dependencia deve estar explicitamente provido usando 'systemPath'

GAV (GroupId / ArtifactId / Version)

Project -> 1.Dependency GAV -> Maven -> 2.Dependency GAV -> Repository (internet) -> 3.Resolved Binary -> Maven -> 4.Resolved Binary -> Project -> 5. Compiled Executable
